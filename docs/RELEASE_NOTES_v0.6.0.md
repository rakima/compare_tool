# compare_tool v0.6.0 Release notes

`.xls` 比較に対応しました。従来の `.xlsx` 比較処理を活かし、Windows版Excelで `.xls` を一時 `.xlsx` に変換してから比較します。

## Highlights

- `.xls` 同士の比較に対応
- `.xls` を一時 `.xlsx` に変換する入力準備層を追加
- `.xls` 比較用サンプル生成に対応
- `.xls` 変換後の一時ファイル削除を強化
- `.xls` 関連の自動テストを強化

## Added

- `.xls` vs `.xls` の比較に対応しました。
  - Windows版Excelを使って `.xls` を一時 `.xlsx` に変換します。
  - 変換後は既存のExcel比較処理で比較します。
  - 比較結果は従来通り `.xlsx` で出力します。
- `.xlsx` と `.xls` のExcel同士比較に対応しました。
  - `.xlsx` はそのまま読み込みます。
  - `.xls` は一時 `.xlsx` に変換してから読み込みます。
- 手動確認用サンプル生成に `.xls` を追加しました。
  - `tools\create_sample_files.py` で `.xlsx` サンプルから `.xls` サンプルを生成します。
  - Excelまたはpywin32が使えない環境では `.xls` サンプルだけスキップします。

## Improved

- `.xls` 変換後の一時ファイルを、比較成功時も失敗時も削除するようにしました。
- `.xls` 変換に失敗した場合のエラー説明を追加しました。
  - Windows版Excelが使えない
  - pywin32が入っていない
  - `.xls` ファイルが破損している
  - `.xls` ファイルがパスワード付き
  - Excelで開けないファイル
- リリースチェックリストに `.xls` の手動確認項目を追加しました。
- サンプル生成スクリプトの出力メッセージを改善しました。

## Tests

- `.xls` 変換失敗時のエラー伝播をテストしました。
- `.xls` 入力が変換済み `.xlsx` として比較処理へ渡されることをテストしました。
- 旧/新どちらも `.xls` の場合に比較できることをテストしました。
- `.xls` 変換後の一時ファイルが、正常終了後に削除されることをテストしました。
- 新ファイル側の変換に失敗した場合でも、旧ファイル側の一時ファイルが削除されることをテストしました。

## Known limitations

- `.xls` 比較にはWindows版Excelが必要です。
- `.xls` 変換にはWindows環境とpywin32が必要です。
- Excelがインストールされていない環境では `.xls` 比較はできません。
- `.xlsm` には未対応です。
- `.xlsx` と `.csv`、`.csv` と `.json`、`.json` と `.xml` のような異なる形式同士の比較には対応していません。
