# compare_tool v0.7.0 Release notes

比較精度とExcel系ファイル対応を強化しました。JSON / XML / Excel / CSVでキー指定による対応付けを改善し、`.xlsm` もExcelブックとして比較できるようになりました。

## Highlights

- JSON配列のキー指定比較に対応
- XML要素の任意キー属性指定に対応
- Excel/CSVキー列比較で1行目のヘッダー名指定に対応
- `.xlsm` 比較に対応
- `.xlsm` サンプル生成に対応
- XMLサンプルに `code` 属性の確認ケースを追加

## Added

- JSONオプションに `配列キー` を追加しました。
  - `id` や `name` などを指定すると、オブジェクト配列をキー値で対応付けます。
  - 配列順が変わっても、同じキーの要素として比較できます。
  - 差分パスは `$.items[id="P002"].price` のように表示します。
  - キー値が重複している場合は、誤対応を避けるため比較を停止します。
- XMLオプションに `要素キー属性` を追加しました。
  - `code` などの属性名を指定すると、同じ親要素内の子要素を属性値で対応付けます。
  - 未指定の場合は従来通り `id` / `name` を自動使用します。
  - キー値が重複している場合は、従来のLCS比較へフォールバックします。
- Excel/CSVのキー列比較で、1行目のヘッダー名を指定できるようにしました。
  - 従来通り `A` / `A,C` の列記号指定も使えます。
  - `商品ID` や `id` のようなヘッダー名でも指定できます。
  - `A1` のようなセル座標は引き続き指定できません。
- `.xlsm` をExcelブックとして比較できるようにしました。
  - `.xlsx` と同じく、そのまま読み込んで比較します。
  - セル値・数式・シート差分を比較します。
  - VBA/マクロは比較対象外です。
  - 比較結果は `.xlsx` として出力します。
- 手動確認用サンプル生成に `.xlsm` を追加しました。
  - `tools\create_sample_files.py` で `.xlsx` サンプルから `.xlsm` サンプルを生成します。
  - Excelまたはpywin32が使えない環境では `.xls` / `.xlsm` サンプルだけスキップします。

## Improved

- Excel/CSVキー列入力欄のUXを改善しました。
  - `キー列で比較` を選んだ時だけ入力欄を有効化します。
  - 説明文を `A / A,C / 商品ID` のように分かりやすくしました。
  - セル座標を指定した場合のエラー説明を改善しました。
- JSON比較結果Excelに `配列キー` の設定を記録するようにしました。
- XML比較結果Excelに `要素キー属性` の設定を記録するようにしました。
- XMLサンプルに `code` 属性で対応付ける確認ケースを追加しました。
  - `XML要素キー属性` に `code` を指定して確認できます。

## Tests

- JSON配列キー指定比較の単体テスト/受け入れテストを追加しました。
- XML任意キー属性比較の単体テスト/受け入れテストを追加しました。
- Excel/CSVキー列ヘッダー名指定のテストを追加しました。
- `.xlsm` 入力の受け入れテストを追加しました。
- `.xlsm` サンプル生成の手動確認を行いました。

## Known limitations

- `.xlsm` のVBA/マクロは比較対象外です。
- `.xlsm` の比較結果は `.xlsx` として出力します。出力ファイルにはVBA/マクロは含まれません。
- `.xls` 比較と `.xls` / `.xlsm` サンプル生成にはWindows版Excelとpywin32が必要です。
- JSON配列キー指定は、指定キーを持つオブジェクト配列で有効です。
- XML要素キー属性は、同じ親要素内の子要素対応付けに使用します。
